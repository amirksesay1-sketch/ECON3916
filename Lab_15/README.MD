The Polynomial Trap: Bias-Variance Tradeoff
This notebook investigates one of machine learning's most fundamental challenges — the bias-variance tradeoff — through controlled experiments on synthetic and real-world data. Using a sine-wave data-generating process (n=50 training, n=200 test, σ=0.3), polynomial regression models of degrees 1–15 were fit and evaluated to isolate how model complexity drives the transition from underfitting to overfitting.
Complexity curves revealed that polynomial degrees 3–5 minimized test RMSE, sitting at the sweet spot where bias and variance balance. Critically, 5-fold cross-validation — implemented both from scratch and via scikit-learn — selected degree 4, closely matching the true test-optimal degree of 5. This confirms CV as a reliable model selection tool even when the ground truth is unknown.
The analysis extended to the Ames Housing dataset, where a parsimonious 5-feature linear model (OverallQual, GrLivArea, GarageCars, GarageArea, TotalBsmtSF) achieved lower CV RMSE (~$41,986) than a kitchen-sink model using all 37 numeric features (~$38,871) — demonstrating that in moderate p/n regimes, feature selection meaningfully reduces variance without sacrificing predictive validity.
Tools: Python, NumPy, scikit-learn (PolynomialFeatures, LinearRegression, cross_val_score), Matplotlib.
Skills demonstrated: bias-variance decomposition, cross-validation, model selection, generalization analysis, polynomial regression.
